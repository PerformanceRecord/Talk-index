# DESIGN.md

## 1. このドキュメントの目的

この `DESIGN.md` は、**Talk-index（トーク索引検索アプリ）** の画面設計を定義します。  
本アプリは「楽曲DB」ではなく、**動画内トークを見出し構造で検索・閲覧するDB** です。

- データソース: Googleスプレッドシート由来の JSON（R2 `index/latest.json`）
- 基本単位: **動画**
- 情報構造: **動画 > 大見出し > 小見出し**（入れ子）

---

## 2. 設計原則

1. **目的先行**: ユーザーが「見たい動画」または「見たいトークテーマ」に最短で到達できること。
2. **二軸探索**: 「動画起点」と「テーマ起点」の両方で探せること。
3. **入れ子の可読性**: 階層（動画/大見出し/小見出し）を崩さないこと。
4. **モバイル優先**: 片手操作でも迷わないUI。

---

## 3. データモデル（表示前提）

README の索引仕様に合わせ、1行を1レコードとして扱います。

- タイトル
- 日付
- URL
- 大見出し
- 大見出しURL
- 小見出し
- 小見出しURL
- 自動検出タグ

UIではこのレコード群を集約して次のツリーに再構成します。

- `video`（動画）
  - `sections[]`（大見出し）
    - `subsections[]`（小見出し）

> 同一動画IDで動画グルーピングし、同一大見出しキーで配下の小見出しを束ねる。

---

## 4. 画面構成

## 4.1 画面A: 動画から探す（動画ビュー）

### 目的

気になる動画を先に選び、その中の話題へ降りる。

### UI要素

- 検索ボックス（動画タイトル / タグ）
- 並べ替え（動画単位）
  - 新しい順 / 古い順
  - タイトル昇順 / 降順
- 動画カード一覧
  - 動画タイトル
  - 日付
  - タグ
  - 大見出し件数
  - 展開時: 大見出し一覧 → 小見出し一覧

### 期待操作

1. 動画を選ぶ
2. 大見出しを開く
3. 小見出しURL（タイムスタンプ付き）へ移動

---

## 4.2 画面B: トークテーマから探す（テーマビュー）

### 目的

気になる話題（大見出し）を先に選び、該当動画へ遷移する。

### UI要素

- 検索ボックス（大見出し / 小見出し / タグ）
- 並べ替え（大見出し単位）
  - 出現数が多い順 / 少ない順
  - 文字列昇順 / 降順
- 大見出しカード一覧
  - 大見出し名
  - 該当動画数
  - 該当小見出し数
  - 展開時: 該当動画ごとの小見出しリンク

### 期待操作

1. テーマを選ぶ
2. 該当動画一覧を見る
3. 目的の小見出しURLへ移動

---

## 4.3 ビュー切替

- 上部タブで `動画から探す / テーマから探す` を切替。
- 切替後も検索文字列は保持（ユーザーの探索文脈を維持）。
- 並べ替え条件はビューごとに保持。

---

## 5. 並べ替え仕様（必須）

本アプリで扱う並べ替えは次の2種類のみ。

1. **動画ごとの並べ替え**（動画ビュー）
2. **大見出しごとの並べ替え**（テーマビュー）

実装時は、どちらの並べ替えが現在有効かを UI 上で明示する（ラベル表示）。

---

## 6. コンポーネント要件

## 6.1 動画カード

- ヘッダ: タイトル / 日付 / タグ
- 本文: 大見出しリスト（折りたたみ可能）
- 子要素: 小見出しリンク（`小見出しURL`）

## 6.2 大見出しカード

- ヘッダ: 大見出し名 / 出現統計
- 本文: 該当動画ごとの小見出しリスト
- 子要素: 動画リンク（`URL`）と小見出しリンク（`小見出しURL`）

## 6.3 検索・絞り込み

- フリーワード検索は以下を対象
  - 動画タイトル
  - 大見出し
  - 小見出し
  - 自動検出タグ
- 0件時は「条件に一致するトークがありません」を表示。

---

## 7. URL遷移仕様

- 動画単位の遷移は `URL` を使う。
- 話題単位の遷移は `大見出しURL` または `小見出しURL` を優先する。
- `?t=` を含むURLはそのまま使い、再計算しない。

---

## 8. 非機能要件

- 初回表示で体感 2 秒以内に一覧を描画（JSONサイズに応じて段階描画可）。
- 不正データ（空見出し、壊れたURL）はスキップし、件数を通知。
- エラー時は「何が不足/失敗したか」を明示する。

---

## 9. 実装時チェックリスト

- [ ] アプリ説明が「楽曲」ではなく「トーク索引」になっている
- [ ] 階層が「動画 > 大見出し > 小見出し」になっている
- [ ] 並べ替えが「動画単位」「大見出し単位」の2種類ある
- [ ] 「気になる動画から探す」「気になるテーマから探す」の両動線がある
- [ ] リンク遷移で `URL / 大見出しURL / 小見出しURL` を使い分けている
